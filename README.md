# 금형 정비 실적 관리 (MOLD MAINTENANCE RECORD)

주간 실적 보고 회의용 PWA 대시보드. `2026년-(보고용)MOLD_MAINTENANCE_RECORD` 엑셀 파일을 브라우저에 업로드하면
대시보드와 시트별 탭(정비실적, 보조자료, 개인별 난이도 수리시간, 금형마스터, 반입반출 이력)으로 정리해서 보여줍니다.

## 실행 방법

가장 간단한 방법: **`dist/index.html`을 더블클릭**하면 서버나 설치 없이 바로 브라우저에서 열립니다.
(`npm run build`로 만들어지는 산출물이며, 원하는 곳으로 복사·이메일 전송·USB로 옮겨도 그대로 동작합니다.)

개발 중이거나 매번 새로 빌드하지 않고 계속 띄워두고 싶다면 `실행.cmd`를 더블클릭해도 됩니다
(최초 1회는 필요한 패키지를 자동 설치하므로 몇 분 걸릴 수 있고, 로컬 서버 + 브라우저가 자동으로 열립니다).

어느 방식이든 엑셀 파일을 업로드한 뒤 "다시 업로드" 버튼으로 최신 파일만 갈아끼우면 됩니다.

데이터는 브라우저(IndexedDB)에 로컬로만 저장되며 외부로 전송되지 않습니다. 서버가 필요 없는 순수 클라이언트 앱입니다.

## 빌드 방식 두 가지

Vite가 만드는 스크립트는 module 방식이라, 크롬이 `file://` 경로에서 이를 CORS로 막아 index.html을 그냥 더블클릭하면
빈 화면만 나오는 문제가 있습니다. 이를 해결하기 위해 빌드 모드를 두 가지로 나눴습니다.

```
npm run build      # dist/index.html 하나로 전부 인라인 (더블클릭용, 기본값)
npm run build:pwa  # 기존 방식 — PWA 설치/오프라인 캐싱 지원, 로컬 서버·호스팅용
```

`build:pwa`로 만든 결과물은 설치 가능한 PWA(서비스워커, 오프라인 캐싱)로 동작하지만 반드시 `http://` 또는
`localhost`로 서빙해야 하며 `file://`로 직접 열면 서비스워커가 등록되지 않습니다. 반대로 기본 `build`는 서비스워커가
없어 PWA 설치는 안 되지만 어디서든 파일 하나로 바로 열립니다. 평소 회의 자료 용도로는 기본 `npm run build`
(더블클릭용)를 쓰고, 설치해서 상시 띄워두고 싶다면 `build:pwa` 결과물을 사내 서버에 올려서 쓰면 됩니다.

## 개발자용 명령어

```
npm install
npm run dev       # 개발 서버
npm run build     # 더블클릭용 단일 파일 빌드 (dist/index.html)
npm run build:pwa # PWA 설치 가능한 빌드 (서버 호스팅용)
npm run preview   # 빌드 결과 미리보기
```

## 데이터 매핑 참고

- **TEK-PM-MOLD RECORD 사용 / 보조자료**: 헤더는 21행, 데이터는 22행부터. 원본의 등급(V열)·PM달성율(X열) 등 일부 컬럼은
  깨진 외부 참조 수식으로 값이 비어있거나 0으로 고정되어 있어 대시보드에서는 제외했습니다.
- **개인별 난이도 수리시간** 탭은 원본의 피벗 테이블을 그대로 옮기지 않고, 사용 시트의 `정비시간` 컬럼을 기준으로
  C(0~2H)/B(2.5~4H)/A(4.5~6H)/S(6.5H~) 등급을 매주 자동 재계산합니다.
- MTTR/MTBF는 일부 행에서 원본 수식 오류로 비정상적인 값(음수, 수백만 분 등)이 섞여 있어 대시보드 KPI는
  평균 대신 중앙값을 사용하고 90일을 초과하는 이상치는 제외합니다. 시트별 상세 테이블에서는 원본 값을 그대로 보여줍니다.
